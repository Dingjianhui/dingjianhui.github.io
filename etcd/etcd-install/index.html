etcd简介

A distributed, reliable key-value store for the most critical data of a distributed system

etcd是一个高可用的键值存储系统
etcd是由CoreOS开发并维护的,灵感来自于 ZooKeeper 等。它使用Go语言编写

Github: https://github.com/etcd-io/etcd
官网:  https://etcd.io

功能特点

简单：基于 HTTP+JSON 的 API 让你用 curl 就可以轻松使用。
安全：可选 SSL 客户认证机制。
快速：每个实例每秒支持一千次写操作。
可信：使用 Raft 算法充分实现了分布式。

主要使用场景

  1. 主要用于共享配置
  2. 服务注册与发现 （Service Discovery）
  3. 分布式锁等

下载etcd并安装

下载地址：https://github.com/etcd-io/etcd/releases
我使用的方式是windows环境下载，然后上传至linux中

[tintin@localhost etcd]$ tar -zxvf etcd-v3.4.9-linux-amd64.tar.gz
[tintin@localhost etcd]$ ls
etcd-v3.4.9-linux-amd64  etcd-v3.4.9-linux-amd64.tar.gz
[tintin@localhost etcd]$ cd etcd-v3.4.9-linux-amd64/
[tintin@localhost etcd-v3.4.9-linux-amd64]$ ls
Documentation  etcd  etcdctl  README-etcdctl.md  README.md  READMEv2-etcdctl.md

创建映射文件 conf data

[tintin@localhost etcd-v3.4.9-linux-amd64]$ cd ../
[tintin@localhost etcd]$ ls
etcd-v3.4.9-linux-amd64  etcd-v3.4.9-linux-amd64.tar.gz
[tintin@localhost etcd]$ mkdir conf
[tintin@localhost etcd]$ mkdir data

下载golang容器
使用golang镜像方面以后有修改源码的时候可重新编译，因为etcd是go编写的
当然也可以直接使用纯净的apline镜像或其他镜像

[tintin@localhost etcd]$docker pull golang:1.14.4-alpine3.12 
[tintin@localhost etcd]$ docker run -it  --name etcd  -p 2379:2379  -v /home/tintin/jtthink/etcd:/etcd golang:1.14.4-alpine3.12 sh
Ctrl+D // 退出容器
[tintin@localhost etcd]$ docker start etcd 
[tintin@localhost etcd]$ ls
conf  data  etcd-v3.4.9-linux-amd64  etcd-v3.4.9-linux-amd64.tar.gz
[tintin@localhost etcd]$ cd etcd-v3.4.9-linux-amd64/
[tintin@localhost etcd-v3.4.9-linux-amd64]$ ls
Documentation  etcd  etcdctl  README-etcdctl.md  README.md  READMEv2-etcdctl.md
[tintin@localhost etcd-v3.4.9-linux-amd64]$ docker cp etcd etcd:/usr/bin && docker cp etcdctl etcd:/usr/bin
[tintin@localhost etcd-v3.4.9-linux-amd64]$ cd ../
[tintin@localhost etcd]$ ls
conf  data  etcd-v3.4.9-linux-amd64  etcd-v3.4.9-linux-amd64.tar.gz
[tintin@localhost etcd]$ cd conf
[tintin@localhost conf]$ vim etcd.yml 创建配置文件

etcd.yml

name: $(hostname -s)
data_dir: /etcd/data
listen-client-urls: http://0.0.0.0:2379

启动etcd

  1. docker exec -it etcd sh (进入容器)
  2. chmod +x /usr/bin/etcd (增加可执行权限)
  3. etcd –version 查看版本
  4. etcd –config-file /etcd/conf/etcd.yml
